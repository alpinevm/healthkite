---
title: Authentication
description: "Bearer token from Settings → Local LAN Server."
---

When **Require auth** is on in the iOS app (default), every endpoint requires:

```http
Authorization: Bearer <token>
```

The token is a 22-character base62 string (~131 bits of entropy) generated by the app and shown in Settings. Tap it to copy.

## Example

```bash
curl -H "Authorization: Bearer oRjUmrdnbrK8reAtGpDIvq" \
     http://192.168.1.244:5606/
```

## Wrong or missing token

```http
HTTP/1.1 401 Unauthorized
Content-Type: application/json; charset=utf-8
WWW-Authenticate: Bearer realm="Wirebody"

{"error":"unauthorized"}
```

## Disabling auth (not recommended)

Toggling **Require auth** off in Settings allows unauthenticated access. The LAN server hot-swaps the auth setting without bouncing the listener — existing connections complete, new requests use the updated auth rule immediately. Use only on a trusted home network.

## Regenerating

Tap **Regenerate Token** in Settings → confirmation alert → fresh 22-char token. Old token invalidates immediately. Update your MCP env (`WIREBODY_TOKEN`) and any other consumers.
