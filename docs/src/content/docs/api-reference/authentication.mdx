---
title: Authentication
description: "TLS-PSK authentication derived from the Wirebody pairing secret."
---

Wirebody's LAN API is not authenticated with an HTTP bearer header. The HTTP endpoints are served inside an authenticated TLS-PSK connection.

The iOS app and `wirebody-mcp` both derive connection material from the **Pairing Secret** shown in Settings:

```text
discovery_id = HKDF-SHA256(root, info = "wirebody:discovery:v1")
psk          = HKDF-SHA256(root, info = "wirebody:auth:v1")
```

- `discovery_id` becomes the Bonjour instance label.
- `psk` is used by TLS-PSK.
- The root secret and PSK are never transmitted.
- `WIREBODY_URL` is unsupported; the MCP server discovers the app with Bonjour/mDNS.

## MCP configuration

```json
{
  "mcpServers": {
    "wirebody": {
      "command": "wirebody-mcp",
      "env": {
        "WIREBODY_TOKEN": "<pairing secret from Wirebody Settings>"
      }
    }
  }
}
```

## Wrong or missing secret

A wrong or missing pairing secret means the MCP server either cannot find the expected Bonjour instance or cannot complete the TLS-PSK handshake. Re-copy the Pairing Secret from Wirebody Settings and restart the MCP client.

## Regenerating

Tap **Regenerate Pairing Secret** in Settings. Old MCP clients stop authenticating immediately and must be updated with the new `WIREBODY_TOKEN` value.
