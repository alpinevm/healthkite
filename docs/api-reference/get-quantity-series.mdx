---
title: GET /quantity/{type}
description: "Standalone HealthKit quantity series in columnar shape."
---

```http
GET /quantity/{type}?from=<iso8601>&to=<iso8601>&limit=5000&offset=0
Authorization: Bearer <token>
```

`type` is a full `HKQuantityTypeIdentifier...` string — get the list from [`/quantity-types`](/api-reference/list-quantity-types). Short forms (`StepCount`) are also accepted when they map unambiguously.

## Query parameters

| Name | Type | Default | Range |
| --- | --- | --- | --- |
| `from` | ISO-8601 UTC | `to - 7 days` | any |
| `to` | ISO-8601 UTC | now | any |
| `limit` | integer | `5000` | 1-50000 |
| `offset` | integer | `0` | ≥ 0 |

The default 7-day window keeps responses bounded; you must opt in to longer ranges.

## Response

```json
{
  "type": "HKQuantityTypeIdentifierStepCount",
  "unit": "count",
  "from": "2026-05-04T00:00:00Z",
  "to":   "2026-05-11T00:00:00Z",
  "sampleEncoding": "columnar-v1",
  "sources": [
    { "id": 0, "device": {...}, "sourceRevision": {...} }
  ],
  "samples": {
    "t":   [0, 60, 120, ...],
    "dur": [60, 60, 60, ...],
    "v":   [12, 8, 15, ...]
  },
  "sampleCount": 1247,
  "limit": 5000,
  "offset": 0,
  "total": 1247,
  "hasMore": false
}
```

Sample-array conventions are the same as for workout streams (see [Columnar encoding](/concepts/columnar-encoding)) with one difference: **`t` is integer seconds offset from `from`** (the response's window start), not from a workout's start.

- `dur` array — included iff any sample has non-zero duration. Step-count and active-energy typically do; HR is instant.
- `src` array — included iff any sample's source differs from `id: 0`.
- `metadata` map — sparse, present iff any sample has metadata.

## Examples

Last week's heart rate samples:

```bash
curl -H "Authorization: Bearer $TOKEN" \
  "http://192.168.1.244:5606/quantity/HKQuantityTypeIdentifierHeartRate?from=2026-05-04T00:00:00Z&to=2026-05-11T00:00:00Z"
```

Resting HR over the past 60 days:

```bash
curl -H "Authorization: Bearer $TOKEN" \
  "http://192.168.1.244:5606/quantity/HKQuantityTypeIdentifierRestingHeartRate?from=2026-03-11T00:00:00Z&to=2026-05-11T00:00:00Z"
```

## Errors

- `404 unknown_type` — identifier doesn't match any known quantity type.
