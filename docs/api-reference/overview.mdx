---
title: API overview
description: "Read-only HTTP/1.1 over LAN, JSON in / JSON out, bearer-token auth."
---

The Wirebody LAN server speaks plaintext HTTP/1.1 over TCP on your home Wi-Fi. Every endpoint is **read-only** and returns `application/json; charset=utf-8`.

## Base URL

The base URL is displayed in the iOS app's **Settings → Local LAN Server** row when the server is on:

```
http://<iPhone-LAN-IP>:<port>
```

Default port is `8080`. The displayed URL uses the iPhone's primary Wi-Fi IPv4 (typically `192.168.x.x`).

## Endpoints

| Method | Path | Purpose |
| --- | --- | --- |
| `GET` | `/` | Status snapshot |
| `GET` | `/workouts` | Paginated workout summaries |
| `GET` | `/workouts/{uuid}` | Full columnar workout detail |
| `GET` | `/quantity-types` | Catalog of standalone HealthKit quantity types |
| `GET` | `/quantity/{type}` | Standalone quantity series, columnar |
| `GET` | `/sleep` | Reconciled nightly sleep sessions |
| `GET` | `/day-snapshot/{date}` | Single-day overview (matches iOS Day view numbers) |

Any other method on these paths returns `405 Method Not Allowed` with `Allow: GET`. Unknown paths return `404 Not Found`.

## Response shapes

All responses share three conventions:

- **HealthKit-native field names.** No invented vocabulary. See [Wire format](/concepts/wire-format).
- **`sampleEncoding: "columnar-v1"`** — discriminator at the top of any response containing sample series. Future versions bump the suffix.
- **`sources` array + `src` indices** — provenance is hoisted. See [Provenance hoisting](/concepts/provenance-hoisting).

## Errors

```json
HTTP/1.1 400 Bad Request
{"error": "bad_request"}

HTTP/1.1 401 Unauthorized
{"error": "unauthorized"}

HTTP/1.1 404 Not Found
{"error": "not_found"}

HTTP/1.1 405 Method Not Allowed
{"error": "method_not_allowed"}

HTTP/1.1 500 Internal Server Error
{"error": "server_error"}
```

The `401` responses also include `WWW-Authenticate: Bearer realm="Wirebody"`.

Query parameters are forgiving: bad `limit`/`offset`/date values clamp to defaults rather than `400`. Path parameters like `/workouts/{uuid}` or `/day-snapshot/{date}` do enforce shape (a malformed UUID won't match anything; a malformed date returns `400 bad_date`).
